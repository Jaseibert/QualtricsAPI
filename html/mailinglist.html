Mailing Lists (XM Subscribers-Only)¶

Working with the methods in the MailingList module, will give you the ability to create create new mailing lists, delete mailing lists, list all of the available mailing lists to your user account, get a Mailing List’s information, rename a Mailing List, list all of the contacts in a mailing list, and create a new contact within a mailing list.

There are several important notes to understand about these functions

This API allows you to create a new contact in the specified mailing list. This API is the only way to add a contact to a mailing list; you cannot copy a contact from the organization’s XM directory to a mailing list.

When you use this API, the contact you create is also created in the organization’s contacts directory. This means that if the contact already exists in the organization’s contacts directory, a duplicate will be created. You can then use XM Directory’s de-duplication tool to remove these duplicate contacts. See Manage Directory Duplicates for more information.

Create a Mailing List¶

MailingList.create_list(name=None)¶

This method will create a mailing list in the XM Directory for the your specified user’s account.

Parameters

list_name – the name of the list to be created.

Returns

set containing the list_name and the list’s new id

Example Implementation¶

Here is an example on how to implement the create_list() method. We will create a new mailing list called ‘MyNewMailingList’ that will be available to your specific user’s account.

#Setup the Credentials if not, already done.
#You only have to do this once.

#Import the module
from QualtricsAPI.XM import MailingList

#Create an instance
m = MailingList()

#Call the method
m.create_list(name='MyNewMailingList')

This will return a tuple containing the new mailing list’s name, and the Mailing List ID associated with the new Mailing List.

List the Mailing Lists Available to your Account¶

MailingList.list_lists(page_size=100, offset=0, to_df=True)¶

This function lists all the mailing lists in the directory for the specified user token.

Parameters

  • page_size – the number of contacts per call.

  • offset – the offset from the order of contacts.

  • to_df – if True, returns the mailing lists and their member objects in a pandas DataFrame.

Returns

either a pandas DataFrame or a list of tuples, containing lists and their respective member objects.

Create a Contact in a Mailing List¶

MailingList.create_contact_in_list(mailing_list=None, first_name=None, last_name=None, email=None, phone=None, external_ref=None, unsubscribed=False, language='en', metadata={})¶

This function creates contacts in a specified mailing list.

Parameters

  • mailing_list (str) – The mailing list id.

  • first_name – the contacts first name.

  • last_name – the contacts last name.

  • email – the contacts email.

  • phone – the contacts phone number.

  • external_ref – the contacts external reference.

  • unsubscribed – denotes whether the contact is unsubscribed.

  • language – the native language of the contact (Default: English)

  • metadata (dict) – any relevant contact metadata.

Returns

the contact id (contact_id) in XMDirectory, and the contact id (contact_list_id) in the mailing list.
